✖️ Multiplicadores

Bônus de XP

Multiplicadores são aplicados sobre o XP base antes de salvar no Firestore. Múltiplos multiplicadores são somados, não multiplicados entre si.

2×
Semana de Boas-Vindas
Primeiros 7 dias após cadastro
1.5×
Plano Elite
Assinante Elite ativo
1.3×
Plano Pro
Assinante Pro ativo
1.2×
Perfil Verificado
Badge "Atleta Verificado" ativo
1.5×
Streak 30 dias
Streak ativa ≥ 30 dias consecutivos
1.25×
Fim de Semana de Destaque
Sábado e domingo — evento especial

Exemplo de cálculo

Atleta Elite + 1ª semana + Streak 30d publica um post:
Base: +20 XP × (1.5 Elite + 2.0 Boas-Vindas + 1.5 Streak30 − 2) = 20 × 3 = +60 XP
Fórmula: XP_final = XP_base × (1 + soma(bonus individuais − 1)). Teto máximo: 3×.

🔥 Streak

Regras de Sequência Diária

A streak é o principal motor de retenção diária. Baseado no modelo Duolingo + Snapchat — o medo de perder é mais poderoso que a recompensa de ganhar.

🔥
Como funciona
Streak conta acesso diário + pelo menos 1 ação (post, laudo, agente, avaliação)
Reinicia à meia-noite horário de Brasília (UTC-3)
Dia perdido = streak zerada (sem exceções automaticamente)
Acesso ao app sem ação não conta como dia de streak
⚡
XP por Streak
Acesso diário válido: +10 XP
Streak 3 dias: bônus +20 XP
Streak 7 dias: bônus +50 XP + badge 🔥
Streak 14 dias: bônus +100 XP
Streak 30 dias: bônus +200 XP + badge 💎 + multiplicador 1.5×
Streak 100 dias: bônus +500 XP + badge lendário 🏆
🛡️
Protetor de Streak (Plano Elite)
Assinante Elite recebe 2 protetores por mês
1 protetor salva 1 dia perdido automaticamente
Protetores não acumulam (máx. 2 ativos)
Plano Free: 0 protetores
Plano Pro: 1 protetor por mês
Inspirado no "Streak Freeze" do Duolingo
😱
Fear of Loss — Alertas
Push às 21h se ainda não completou ação do dia
Push às 23h com urgência máxima ("Faltam 60 min!")
Mensagem do Agente IA personalizada por personalidade
Notificação in-app com banner vermelho pulsante
🎯 Missões

Sistema de Missões

3 camadas de missões com dificuldade e recompensa crescentes. Inspirado em Nike Run Club + Habitica + Chess.com.

☀️
Missões Diárias
3 missões por dia, renovam à meia-noite (Brasília)
Completar todas as 3: bônus +30 XP extra
Missões fixas + 1 missão aleatória rotativa
Missões fixas: Abrir app (+10), Interagir com Agente (+20), Visualizar perfil (+15)
Rotativas (aleatorias): Curtir 3 posts, Responder comentário, Ver laudo existente
📅
Missões Semanais
5 missões por semana, renovam toda segunda-feira 00h
Completar todas as 5: bônus +150 XP extra
Missão não concluída expira e não acumula
Publicar 3 posts (+90), Gerar 1 Laudo Scout (+120), Adicionar 1 vídeo (+80), Solicitar 1 avaliação (+150), Acessar 5 dias seguidos (+200)
🗓️
Desafios Mensais
2-3 desafios por mês, renovam no dia 1 às 00h
Cada desafio entrega um badge exclusivo + XP alto
Desafios são progressivos (não binários)
Badge do mês aparece no perfil com a data
Scout Ready (3 laudos → +400 XP 🎬), Atleta Visível (100 visitas → +300 XP 👁️), Rede Conectada (10 conexões → +500 XP 🤝)

Missões de Evento (Especiais)

Ex: "Semana do Futebol Feminino", "Copa SP", "Janela de transferências"

Badge comemorativo exclusivo por período
Configuradas via Firestore pelo painel admin

Competição semanal segmentada por categoria e posição. Baseado em Duolingo + Strava — o ranking é o principal motor de competição.

🏆
5 Ligas (do menor ao maior)

🥈 Prata — intermediário
🥇 Ouro — competitivo
💚 Esmeralda — avançado

Regras de Promoção
Ranking reseta toda segunda-feira às 00h

Últimos 3 da liga descem para a anterior (exceto Bronze)
Liga Bronze: ninguém desce (piso do sistema)
Empate desempata por quem atingiu o XP primeiro

Recompensas de Liga

🛡️ Anti-Abuso

Regras de Proteção do Sistema

Limites para evitar farm de XP e garantir que as recompensas reflitam engajamento real.

RegraDescriçãoAção ao violar
Rate limit diário de postsMáx. 3 posts com XP por dia. Posts adicionais publicam mas não geram XP.XP bloqueado, post publicado normalmente
Cooldown de laudo scoutMínimo 24h para gerar novo laudo do mesmo vídeoBotão desabilitado com timer
Visualização de próprio perfilAcessar o próprio perfil não gera XP de "visita"Flag userId === viewerId
Auto-comentárioComentar no próprio post não conta como receber comentárioFlag no Firestore
Mesma ação repetida no diaMissão diária concluída não conta novamente até o resetMissão marcada como done no Firestore
Multi-contaIP + device fingerprint: máx. 2 contas distintas por dispositivoXP suspenso, revisão manual
Mesmo avaliador repetidoMesmo avaliador só pode gerar XP para o mesmo atleta a cada 30 diasXP bloqueado, avaliação salva sem pontuação
Visita repetida de clubeMesmo clube só gera XP de visita 1× a cada 48hFlag de cooldown por clubId no Firestore
XP suspeito (spike)Ganho de XP > 500 em 1h dispara revisão automáticaXP congelado, notificação ao admin

⚙️ Arquitetura

Implementação Firebase

O motor de XP roda em Cloud Functions com Firestore como fonte de verdade. Todas as operações de XP são atômicas para evitar inconsistências.

1
Ação do usuário no app (Angular)
Usuário realiza ação (post, laudo, avaliação, etc). O serviço Angular chama a Cloud Function via HTTPS callable.
functions.httpsCallable('awardXP')({ action: 'post', userId: uid })
2
Cloud Function: awardXP
Valida a ação, verifica rate limits e cooldowns no Firestore, aplica multiplicadores, usa transação atômica para somar o XP.
Firestore: users/{uid} → xp, level, streak, badges[]
3
Verificação de nível e badges
Após somar XP, a função verifica se o novo total cruza algum threshold de nível ou badge. Se sim, dispara o desbloqueio.
if (newXP >= LEVEL_THRESHOLDS[currentLevel + 1]) → levelUp()
4
Atualização de missões
Verifica progresso de missões ativas (diária, semanal, mensal). Se missão completada, soma bônus e marca como done.
Firestore: missions/{uid}/{missionId} → progress, completed, expiresAt
5
Atualização do ranking da liga
Após somar XP, atualiza o documento do ranking semanal por categoria. Ranking é calculado em tempo real via query ordenada.
Firestore: leaderboard/{category}/{weekId} → [{uid, xp, level, name}]
6
Disparo de notificações (FCM)
Triggers automáticos via Firestore onChange + Cloud Scheduler para notificações temporizadas (streaks, resumo semanal, etc.).
Cloud Scheduler: cron "0 21 * * *" → checkStreakRisk()
7
Reset semanal de liga (Scheduled Function)
Toda segunda-feira às 00h, calcula promoções/rebaixamentos, cria novo documento de semana, notifica atletas afetados.
Cloud Scheduler: cron "0 0 * * 1" → weeklyLeagueReset()
